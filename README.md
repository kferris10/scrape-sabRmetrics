# scrape-baseball

> This README was generated by AI (Claude).

An R pipeline that scrapes MLB and minor league baseball data from multiple sources and stores it in PostgreSQL or BigQuery. Supports daily automated scrapes and full historical backfills.

## What it collects

| Data | Source | Coverage |
|------|--------|----------|
| Game schedule & scores | MLB Stats API | MLB, AAA, AA, A+, A |
| Play-by-play events | MLB Stats API | MLB, AAA, AA, A+, A |
| Pitch-level data (location, spin, velocity) | MLB Stats API | MLB, AAA, AA, A+, A |
| Statcast pitch metrics (90+ columns) | Baseball Savant | MLB only |
| Player biographical info | MLB Stats API | All levels |
| Season hitting/pitching stats | MLB Stats API | All levels |

## Prerequisites

- R 4.4+
- PostgreSQL 17 (postgres backend) or a GCP project with BigQuery enabled (bigquery backend)

## Setup

**1. Install R dependencies**

```bash
Rscript -e 'renv::restore()'
```

**2. Create a `.env` file** in the project root:

For PostgreSQL:
```
BASEBALL_DB_PASSWORD=your_password_here
```

For BigQuery:
```
GCP_PROJECT_ID=your_gcp_project_id
BQ_DATASET=baseball
SCRAPE_BACKEND=bigquery
```

**3. Initialize the database** (PostgreSQL only)

```bash
psql -U postgres -d baseball -f sql/schema.sql
psql -U postgres -d baseball -f sql/views.sql
```

BigQuery tables and views are created automatically on first run via `ensure_tables()`.

The PostgreSQL connection defaults to `localhost:5432`, database `baseball`, user `postgres`. Edit `config.yml` to change these.

## Usage

### Daily scrape (yesterday's games)

```bash
Rscript R/run_scrape.R --mode daily
```

### Daily scrape for a specific date

```bash
Rscript R/run_scrape.R --mode daily --date 2025-06-15
```

### Backfill a full season

```bash
Rscript R/run_scrape.R --mode backfill --year 2024
```

### Backfill a date range

```bash
Rscript R/run_scrape.R --mode backfill --start 2024-03-01 --end 2024-09-30
```

### Player biographical data

```bash
Rscript R/run_scrape.R --mode players --year 2024
```

### Season summary stats (hitting + pitching)

```bash
Rscript R/run_scrape.R --mode season-summary --year 2024
```

### Retry a failed scrape

```bash
# Find the failed log ID first, then retry it
Rscript R/run_scrape.R --mode retry --log-id 42
```

### Additional flags

```bash
# Only scrape specific levels (default: MLB, AAA, AA, A+, A)
Rscript R/run_scrape.R --mode daily --levels MLB,AAA

# Only run specific scrapers
Rscript R/run_scrape.R --mode daily --sources schedule,statsapi

# Filter game types (S=spring, R=regular, D/L/W/C=playoffs)
Rscript R/run_scrape.R --mode backfill --year 2024 --game-types R

# Disable parallel processing
Rscript R/run_scrape.R --mode backfill --year 2024 --no-parallel

# Use BigQuery backend
Rscript R/run_scrape.R --mode daily --backend bigquery
```

## Database schema

Eight tables in PostgreSQL:

- **`schedule`** — one row per game (teams, scores, date, level)
- **`statsapi_event`** — one row per plate appearance (batter, pitcher, outcome)
- **`statsapi_pitch`** — one row per pitch (location, movement, velocity)
- **`statsapi_play`** — one row per runner/play action (stolen bases, pickoffs, baserunner state)
- **`statcast`** — one row per pitch with full Statcast payload stored as JSONB (90+ columns)
- **`player`** — biographical data (name, DOB, handedness, position)
- **`season_summary`** — season-level hitting and pitching stats, stored as JSONB
- **`scrape_log`** — tracks every scrape run (status, game counts, errors)

### Analytical view

`mv_statcast` (PostgreSQL) and `v_statcast` (BigQuery) unpack ~25 typed columns from the Statcast JSON payload into queryable columns (pitch type, velocity, launch speed/angle, expected wOBA, win expectancy, etc.) and join player names. The PostgreSQL materialized view refreshes automatically after every daily, backfill, or retry run; the BigQuery view is a live SQL view.

```sql
-- Example: all four-seamers thrown by a pitcher this season
SELECT game_date, batter_name, release_speed, pfx_x, pfx_z, events
FROM mv_statcast
WHERE pitcher_name = 'Gerrit Cole'
  AND year = 2024
  AND pitch_type = 'FF'
ORDER BY game_date;
```

## Project structure

```
R/
  run_scrape.R           # CLI entry point
  db.R                   # PostgreSQL backend (connection, upserts, view refresh)
  db_bigquery.R          # BigQuery backend (same public API as db.R)
  utils.R                # Logging, date chunking, parallel cluster
  scrape_schedule.R      # Game schedule/results
  scrape_statsapi.R      # Event/pitch/play data
  scrape_baseballsavant.R# Statcast data (MLB only)
  scrape_players.R       # Player biographical data
  scrape_season_summary.R# Season stats
sql/
  schema.sql             # PostgreSQL table definitions (idempotent)
  views.sql              # Indexes + mv_statcast materialized view
config.yml               # DB connection and scraping settings
```

## Configuration

Edit `config.yml` to change defaults:

```yaml
database:
  host: "localhost"
  port: 5432
  dbname: "baseball"
  user: "postgres"

scraping:
  levels: [MLB, AAA, AA, A+, A]
  parallel_cores: 4
  chunk_days: 7       # backfill processes this many days at a time
```

The database password is always read from the `BASEBALL_DB_PASSWORD` environment variable (or `.env` file) — never hardcoded.
